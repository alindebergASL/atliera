# Atliera Legacy Comparison Protocol

Atliera is a fresh system. Legacy account-research outputs are comparison references only, not runtime dependencies.

## Purpose

Use this protocol when comparing old account-research reports against fresh Atliera output.

## Inputs

- Old report URL or authenticated legacy route.
- Atliera account/workshop URL.
- Account name/domain.
- Research run ID if available.

## Comparison categories

1. Recovered themes
   - Important legacy themes Atliera also found with evidence.

2. New findings
   - Useful Atliera findings that were missing from legacy reports.

3. Missed but potentially valuable legacy insights
   - Legacy points Atliera did not capture, requiring targeted re-research or product-gap review.

4. Unsupported/outdated legacy claims
   - Legacy prose that lacks source support, appears stale, or should not be promoted.

5. Evidence quality delta
   - Whether Atliera has better source/excerpt/claim lineage than the legacy report.

6. Workshop usefulness delta
   - Whether Atliera Workshop is more useful than the legacy report for understanding the account and deciding next steps.

## Hard rule

Do not import, verify, or promote legacy prose automatically. Legacy comparison can trigger targeted re-research only.
